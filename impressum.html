Das SBKIM Such-Werkzeug ist eine server-lose Anwendung: Es existiert kein eigener Anwendungs-Server, kein Nutzerkonto und keine Analyse- oder Tracking-Software. Es werden keine personenbezogenen Daten an den Betreiber übertragen.

Hosting

Diese Seite wird über GitHub Pages (GitHub Inc.) ausgeliefert. Beim Aufruf werden technisch notwendige Zugriffsdaten (z. B. IP-Adresse, Browser-Angaben) an GitHub übertragen, um die Seite auszuliefern; es gelten deren Datenschutzbestimmungen.

Was lokal im Browser bleibt

  • Such-Einstellungen (Position, Größe, gewählte Suchmaschine, EU-Politik) werden in localStorage gespeichert — nur auf deinem Gerät.
  • Ein optionaler API-Schlüssel (für Sprach-Cloud oder KI-Richter) wird verschlüsselt in einem lokalen Tresor gespeichert (PBKDF2 + AES-GCM, Wiederherstellung über Shamir 2-von-3). Er verlässt dein Gerät nur in Richtung des von dir gewählten Anbieters — nie an den Betreiber, nie an Dritte.

Verbindungen zu Dritten (nur auf deine Aktion)

  • Embedding-Modell: Beim ersten Gebrauch der semantischen Suche wird ein KI-Modell einmalig von einem öffentlichen CDN (jsDelivr / Hugging Face) geladen. Dabei gelten deren Datenschutzbestimmungen.
  • KI-Anbieter (ChatGPT, Claude, Gemini, Perplexity): nur, wenn du den KI-Pfad bewusst auslöst. Deine Anfrage geht dann an den gewählten Anbieter; dessen Datenschutz gilt.
  • Sprach-Cloud (optional, EU-Endpunkt): nur, wenn du die Cloud-Spracheingabe mit eigenem Schlüssel nutzt.
  • Web-Suche „↗ neuer Tab": öffnet die von dir gewählte Suchmaschine in einem neuen Tab — das Werkzeug selbst lädt nichts.

Es findet keine eigenständige Datensammlung statt. Jede Außenverbindung ist benannt, sichtbar und durch dich ausgelöst.
